# 시스템 보고서: 미국-한국 주식 연계 자동매매 시스템

> 작성일: 2026-04-04
> 버전: 1.2
> 상태: 모의투자 운영 중

---

## 1. 시스템 개요

미국 증시(선행)와 한국 증시(후행)의 상관관계를 분석하여 시장 국면을 자동 판단하고,
전략과 가중치를 조정하며 KIS API를 통한 모의투자 자동매매를 수행하는 멀티에이전트 시스템.

### 배포 구조

| 구분 | 호스팅 | 실행 방식 |
|------|--------|----------|
| Python 백엔드 | GCP VM | PM2 (`python main.py --mode schedule`, 30분 주기) |
| Next.js 프론트엔드 | Vercel | GitHub push 시 자동 배포 |
| 데이터베이스 | Supabase | PostgreSQL (실시간 구독 포함) |

### 기술 스택

| 항목 | 기술 |
|------|------|
| 백엔드 | Python 3.11 (async/await) |
| 프론트엔드 | Next.js + TypeScript + Tailwind CSS |
| DB | Supabase (PostgreSQL + Realtime) |
| 주문 API | 한국투자증권 KIS API (모의투자) |
| 미국 데이터 | yfinance |
| 한국 데이터 | pykrx + KIS API |
| 차트 | Recharts |
| 알림 | Telegram Bot API |

## 2. 실행 흐름

### 2.1 실행 모드

```bash
python main.py                    # 1회 실행
python main.py --mode schedule    # 30분 주기 반복 (GCP VM 기본)
python main.py --mode update-history  # 최근 30일 데이터 갱신
python main.py --mode fetch-history   # 5년치 전체 재수집
```

### 2.2 파이프라인 (1회 실행 기준)

```
[사전 체크] KIS 토큰 검증 → 일간/주간 손실한도 확인
     │
     ▼
[Step 1] 데이터수집 (DC)
  미국: 나스닥, S&P500, SOX, VIX, 환율, 선물, NVDA/AMD/TSLA
  한국: KOSPI, KOSDAQ, 개별종목 (반도체/2차전지/정유 등)
  원자재: WTI, 금, 구리, 리튬
     │
     ▼
[Step 2] 시장분석 (MA) + 이슈감지 (IM) — 병렬 실행
  → 6단계 국면 판별 + 선행지표 신호 + 이슈 탐지
     │
     ▼
[Step 2b] 보유 포지션 검토 + 청산 계획 수립
     │
     ▼
[Step 3] 가중치조정 (WA)
  → 국면 + 이슈 반영 → 공격/방어/현금 비중 결정 → 종목 선정
     │
     ▼
[Step 4] 전략적용 (SR)
  → 전략 라이브러리에서 적합 전략 매칭 → 진입/제외 조건 필터링
     │
     ▼
[Step 5] 주문실행 (EX)
  → KIS API 매수/매도 → 포지션 DB 기록 → 텔레그램 알림
```

### 2.3 세션 기반 스케줄링 (장중 시간대별 동작)

| 시간 | 세션 | 동작 |
|------|------|------|
| 06:00~08:30 | PRE_ANALYSIS | 분석만 수행, 매수 불가 |
| 08:30~09:05 | MARKET_OPEN_WAIT | 동시호가 대기 |
| 09:05~09:30 | ENTRY_READY | 시장 안정화 확인 |
| 09:30~15:10 | NORMAL | 정상 매매 |
| 15:10~15:30 | CLOSING | 초단기 강제 청산 |
| 15:30~06:00 | AFTER_HOURS | 파이프라인 중단, 분석만 |

### 2.4 독립 모니터링 루프 (파이프라인과 별도)

- **손절/익절 감시**: 변동성 높은 국면 1분, 일반 3분 간격
- **디버거**: 전 에이전트 헬스체크 (30초 HEARTBEAT)
- **일간 백테스트**: 장 종료 후 자동 실행

| 일반장 | -3% ~ +3%, VIX < 20 | 60% | 0% | 40% | 단기 |

## 6. 리스크 관리

### 6.1 손실 한도

| 구분 | 한도 | 동작 |
|------|------|------|
| 일간 손실한도 (DSL) | -3% | 신규 매수 중단 |
| 주간 손실한도 (WSL) | -5% | 신규 매수 중단 |
| 연속 손실 | 3회 | 공격 배분 × 0.5 |
| 회복 모드 | DSL 발동 + 상승 추세 | 30% 규모로 제한 진입 허용 |

### 6.2 포지션 한도 (국면별)

| 국면 | 최대 동시 보유 |
|------|--------------|
| 대상승장 | 5개 |
| 상승장 | 4개 |
| 일반장 | 3개 |
| 변동폭큰 | 2개 |
| 하락장 | 1개 |
| 대폭락장 | 1개 |

### 6.3 보유기간별 청산 기준

| 기간 | 보유 | 익절 | 손절 | 트레일링 스탑 |
|------|------|------|------|-------------|
| 초단기 | ~3시간 | +1.5% | -0.7% | 없음 (15:20 강제청산) |
| 단기 | 1~5일 | +10% | -2% | +1.5% (활성: +2%) |
| 중기 | 1~20일 | +15% | -3% | +2.5% (활성: +3%) |
| 장기 | 1~90일 | +25% | -5% | +4% (활성: +8%) |

### 6.4 분할 매수/매도

**DCA (분할 매수):**
- 1차 진입: 60%
- 2차 진입: 40% (가격 -1% 하락 시, 4시간 내)
- 4시간 후 자동 취소

**분할 익절:**
- +1.5% 도달: 30% 매도
- +3.0% 도달: 30% 매도
- 나머지: 트레일링 스탑

### 6.5 청산 우선순위

```
1. 손절 (SL) — 즉시
2. 초단기 시간 청산 (15:20) — 당일 강제
3. 트레일링 스탑 — 고점 대비 하락
4. 최대 보유일 초과
5. 국면 전환 (PHASE_CHANGE)
6. 신호 역전 (초단기/단기만)
```

## 10. 통신 프로토콜

모든 에이전트 간 통신은 `StandardMessage` 형식을 사용합니다.

```
DC → PP: RAW_MARKET_DATA (단방향)
PP → MA: PREPROCESSED_DATA (단방향)
PP → IM: PREPROCESSED_DATA (단방향)
MA → WA: MARKET_ANALYSIS (단방향)
IM → WA: ISSUE_ANALYSIS (단방향)
WA → SR: SIGNAL (양방향)
SR → EX: SIGNAL (단방향)
EX → DB: ORDER (단방향)
모든 에이전트 → DB: 감시용 (단방향)
모든 에이전트 → OR: 보고용 (단방향)
```
