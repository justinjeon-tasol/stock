## 2. 핵심 약점 7가지

### 약점 1: 총 자산 리스크 관리의 부재 (치명적)

**현재 상태:**
- 단일 종목 최대 비중 15% 캡만 존재 (max_stock_weight_pct: 0.2)
- **일일 최대 손실 한도(Daily Stop Loss)가 없음**
- **연속 손실 시 포지션 축소 로직 없음** (마틴게일 효과)
- **총 노출(Gross Exposure) 한도 없음**

**위험:**
한 번의 급락에서 보유 종목 5개가 동시에 -3% 하락하면 총 자산 대비 -9% 이상 손실 가능.
손절 트리거가 작동하기 전에 갭 하락(Gap Down)이 발생하면 예상보다 훨씬 큰 손실.

**실전 시나리오:**
```
상승장 → 공격 비중 80%
  종목 A(반도체) -3%, 종목 B(반도체) -4%, 종목 C(2차전지) -2%
  → 하루 손실: 약 -7.2% (총자산 대비)
  → 연속 2일: -14% → 원금 회복에 +16.3% 필요
```

---

### 약점 2: 매수 진입 로직의 단순함 (높음)

**현재 상태:**
- _place_order()에서 **시장가 일괄 매수** (ORD_DVSN: "01")
- 분할 매수(DCA) 전략 없음
- 진입 가격 확인(호가 스프레드) 없음
- 거래량 대비 주문 수량 체크 없음 (소형주 유동성 리스크)

**위험:**
- 시장가 주문은 호가 스프레드가 넓은 종목에서 불리한 체결
- 장 시작 직후 변동성이 큰 시점에 전량 매수 → 고점 매수 리스크
- 한미반도체(042700) 같은 중소형주는 일일 거래 대금 대비 매수 금액이 과도할 수 있음

---

### 약점 3: 종목 유니버스의 편중 (높음)

**현재 유니버스 (strategy_config.json):**
- 총 16종목, 8섹터
- 반도체 3종목이 가장 많고, 선행지표 매핑도 반도체에 집중
- **역선행(Contrarian) 종목 없음** (하락장 방어 종목)
- **인버스/레버리지 ETF 없음** (하락장 수익 기회 포기)
- 방어주(통신, 유틸리티, 필수소비재) 부재
- defensive 비중이 정의되어 있지만 실제 매수 대상이 없음

---

### 약점 4: 청산 로직의 맹점 (높음)

**현재 상태:**
- 손절: 고정 퍼센트 (-0.5% ~ -7.0%)
- 익절: 고정 퍼센트 (+0.8% ~ +20.0%)
- 트레일링 스탑: 중기/장기만 적용
- **장중 급등 후 하락 패턴(갈고리형) 대응 없음**
- **부분 익절(Partial Take Profit) 없음**

**위험:**
```
단기 종목: 삼성전자 매수 후 +2.5% 도달 → 전량 익절
  → 이후 +8%까지 상승 → 수익 기회 +5.5% 포기

+3%에서 절반을 미리 실현했다면: 평균 +4.5% 확보 + 나머지 추세 추종
```

---

### 약점 5: 선행지표 매핑의 한계 (중간)

**현재 상태:**
- **미국 상승 → 한국 상승** 방향만 매핑
- **미국 하락 → 한국 하락** 방향의 체계적 활용 없음
- SOX -3% 하락 시 반도체 매도 신호가 이슈감지(ISS_005)에만 의존
- 미국-한국 **시차 활용**이 체계적이지 않음 (장전/장중/장후 구분 없음)

**위험:**
미국 나스닥 -3% 급락 → 다음날 한국 코스닥 -2~4% 예상되지만,
현재 시스템은 BUY 신호 부재 → HOLD만 하고,
기존 보유 종목의 **선제적 청산**은 국면 전환(3일차+)이 되어야 발동.
1~2일 동안의 낙폭을 고스란히 흡수함.

---

### 약점 6: 성과 피드백 루프의 부재 (중간)

**현재 상태:**
- trades 테이블에 매매 기록 저장
- account_summary에 잔고 스냅샷 저장
- **하지만 이 데이터를 분석하여 전략을 조정하는 자동 피드백 루프가 없음**
- 전략연구(strategy_researcher.py)는 히스토리 데이터 백테스팅만 수행

**위험:**
- 특정 전략이 실전에서 계속 손실을 내도 자동 비활성화되지 않음
- 특정 섹터에서 반복 손실 패턴이 있어도 유니버스에서 자동 제외되지 않음
- 시장 구조 변화(레짐 시프트)에 적응하지 못함

---

### 약점 7: 시장 미시구조(Microstructure) 무시 (중간)

**현재 상태:**
- 파이프라인이 30분 주기로 실행
- 장 시작 09:00~09:30 변동성 구간 구분 없음
- 동시호가(08:30~09:00, 15:20~15:30) 고려 없음
- **초단기 전략에서 장중 1분/3분 청산만 있고, 진입 타이밍 제어 없음**

## 3. 개선 방안 상세

### 개선 1: 다층 리스크 관리 시스템 도입 (치명적 → 최우선)

> [!CAUTION]
> **이것이 가장 중요합니다.** 리스크 관리 없이 수익 전략을 아무리 정교하게 만들어도 한 번의 블랙스완으로 원금을 잃습니다.

#### 1-1. Daily Stop Loss (일일 최대 손실 한도)

```json
// config/risk_config.json (신규)
{
  "daily_stop_loss": {
    "max_daily_loss_pct": -3.0,
    "max_weekly_loss_pct": -5.0,
    "cooldown_hours": 24,
    "action": "HALT_NEW_ENTRY"
  }
}
```

- 일일 최대 손실 -3% 도달 시 당일 매매 중단 (신규 진입만, 기존 포지션 청산은 계속)
- 주간 최대 손실 -5% 도달 시 해당 주 매매 중단
- 발동 후 최소 24시간 대기

**구현 위치:** orchestrator.py — run_once() 시작 시점에 체크

#### 1-2. 연속 손실 감지 및 포지션 축소

```json
{
  "consecutive_loss": {
    "trigger_count": 3,
    "reduce_aggressive_by": 0.5,
    "recovery_condition": "1_win"
  }
}
```

- 연속 3회 손절 발생 시 공격 비중 50% 축소
- 1회 수익 실현 후 원래 비중 복원

**구현 위치:** weight_adjuster.py — _decide_weight_config() 내부

#### 1-3. 최대 동시 포지션 수 제한

```json
{
  "max_positions": {
    "대상승장": 5,
    "상승장": 4,
    "일반장": 3,
    "변동폭큰": 2,
    "하락장": 1,
    "대폭락장": 1
  }
}
```

현재 _MAX_TARGETS = 5 (고정) → 국면에 따라 동적으로 제한

#### 1-4. 상관관계 기반 포트폴리오 리스크

같은 섹터 종목이 이미 보유 중이면 추가 매수 비중 감소.
예: 삼성전자(반도체) 보유 중 + SK하이닉스(반도체) 매수 신호 → SK하이닉스 weight × 0.5

### 개선 2: 분할 매수/매도 전략 도입

#### 2-1. 분할 매수 (DCA)

```json
{
  "dca_config": {
    "enabled": true,
    "splits": 2,
    "split_ratio": [0.6, 0.4],
    "second_entry_condition": {
      "type": "pullback",
      "pullback_pct": -1.0,
      "max_wait_hours": 4
    }
  }
}
```

- 1차 매수: 예산의 60%
- 2차 매수: 1차 매수가 대비 -1% 하락 시 나머지 40%
- 2차 미발동 시: 나머지 예산 현금 유지 → 다음 파이프라인에서 재평가

**구현 위치:** executor.py

#### 2-2. 분할 매도 (Partial Take Profit)

```json
{
  "partial_take_profit": {
    "enabled": true,
    "levels": [
      {"pnl_pct": 1.5, "sell_ratio": 0.3},
      {"pnl_pct": 3.0, "sell_ratio": 0.3},
      {"remaining": "trailing_stop"}
    ]
  }
}
```

**효과:**
- 수익 확정 구간 확보 → 심리적 안정 + 실현 수익 축적
- 나머지 물량은 추세 추종 → 대형 상승 참여 가능
- 전량 익절 대비 평균 수익률은 소폭 낮지만, **수익 일관성(Consistency)이 크게 향상**

### 개선 3: 종목 유니버스 확장 및 방어 전략

#### 3-1. 방어 자산 추가

```json
{
  "방어ETF": [
    {"code": "148070", "name": "KOSEF 국고채10년"},
    {"code": "132030", "name": "KODEX 골드선물(H)"},
    {"code": "261240", "name": "KODEX 미국채울트라30년선물(H)"}
  ],
  "인버스ETF": [
    {"code": "114800", "name": "KODEX 인버스"},
    {"code": "252670", "name": "KODEX 200선물인버스2X"}
  ]
}
```

#### 3-2. 국면별 방어자산 활용

```
하락장/대폭락장에서 defensive 비중:
  기존: 방어 전략 40% → 실제로 무엇을 사는지 정의 안 됨
  개선: 방어 전략 40% → 국고채 ETF 20% + 골드 ETF 10% + 현금 10%
```

**현재 문제:** phase_weights에 defensive 비중은 정의되어 있지만,
_decide_targets()에서 **defensive 자산에 실제 매수 주문을 내리지 않음**.
defensive 비중이 사실상 현금과 동일하게 작동하고 있음.

### 개선 4: 선행지표 양방향 활용 → 선제적 청산

#### 4-1. 미국 하락 → 한국 선제 대응

market_analyzer.py에 하락 방향 매핑 추가:

| 신호 | 임계값 | 한국 대응 |
|------|--------|----------|
| nasdaq_crash | -2.0% | 지수ETF 매도 |
| sox_crash | -3.0% | 반도체 매도 |
| vix_warning | 25 이상 | 전체 포지션 50% 축소 |
| tesla_crash | -3.0% | 2차전지 매도 |

#### 4-2. 시차 활용 체계화

```
미국 장 마감 (한국 시간 06:00)
  ├── 미국 급등 → 09:05 이후 진입 (장 시작 변동성 회피)
  ├── 미국 급락 → 09:00 동시호가에 시장가 매도 (선제 청산)
  └── 미국 보합 → HOLD 유지
```

**구현:** main.py의 schedule 모드에 시간대별 파이프라인 모드 추가
- 06:00~08:30: 사전분석 모드 (미국 마감 데이터 기반 전략 결정)
- 09:05~09:30: 매수 실행 모드 (장 시작 후 안정화 확인 후 진입)
- 15:10~15:20: 초단기 청산 모드 (장 마감 전 정리)

### 개선 5: 성과 피드백 및 전략 자가진화

#### 5-1. 전략 자동 비활성화

```
비활성화 조건 (3개 중 2개 충족 시):
- 실전 승률이 백테스트 승률의 70% 미만
- 최근 10건 중 연속 3건 이상 손실
- 실전 MDD가 백테스트 MDD의 1.5배 초과
```

구현: agents/performance_reviewer.py (신규) 또는 strategy_researcher.py 확장

#### 5-2. 시장 적응형 임계값 조정

고정 임계값 → 히스토리 기반 동적 임계값:
- SOX threshold_pct 2.0% (고정) → 최근 20일 SOX 일일 변동성 기준 1.5시그마 (동적)
- VIX 30 → 최근 1년 VIX 분포에서 상위 10% (동적)

#### 5-3. 수익률 기반 전략 비중 재배분

- 최근 30일 실전 성과 기반 전략 비중 자동 조정
- 성과 좋은 전략 → 비중 확대 (최대 2배)
- 성과 나쁜 전략 → 비중 축소 (최소 0.3배)
- Kelly Criterion 변형 적용

### 개선 6: 주문 실행 품질 개선

#### 6-1. 지정가 주문 도입

```
현재: 시장가 주문만 사용 (ORD_DVSN: "01")
개선: 현재가 - 0.1% (1호가 유리하게 매수) → 지정가 주문
미체결 시: 일정 시간 후 시장가로 전환
```

#### 6-2. VWAP 기반 진입

장 시작 후 30분 VWAP(거래량 가중 평균가) 이하일 때만 매수
→ "비싸게 사지 않겠다"는 원칙

#### 6-3. 유동성 체크

일일 평균 거래대금 대비 주문 금액 비율 제한:
