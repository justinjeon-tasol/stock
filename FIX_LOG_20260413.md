## 근본 원인 (3가지 복합)

### 1. EXIT_PLAN trigger_price가 매입가 미만에 설정됨
- `PROFIT_FLAT`: `s1 = max(avg_price * 1.005, current_price * 0.995)` → current_price * 0.995는 현재가보다 낮음
- `LOSS_ZONE`: `s1 = max(current_price * 1.02, avg_price * 0.99)` → avg_price * 0.99 = 매입가 -1%에서 발동 가능
- 결과: 손실 상태에서 "익절" 실행이라는 모순

### 2. sell_targets 경로에 보호 시간 없음
- WeightAdjuster가 SIGNAL_EXIT로 매도 지시하면 보유 1분이어도 즉시 실행
- `_check_stop_take`에는 2시간 보호가 있지만 sell_targets에는 없었음

### 3. 쿨다운 체크 실패 시 `pass`로 무시
- Supabase 조회 실패 시 쿨다운 4시간이 무효화 → 손절 직후 재매수 허용

## 수정 내용

### 수정 1: DCA/동기화 trades 기록 누락 (커밋 1fd4a63)
- **executor.py**: DCA 2차 매수 체결 시 `save_trade()` 추가
- **orchestrator.py**: KIS 동기화 BUY/SELL 시 `save_trade()` 추가

### 수정 2: EXIT_PLAN 손실 상태 익절 방지
- **executor.py `_check_exit_plan()`**: PARTIAL_TP는 현재가 >= 매입가일 때만 실행
- FINAL_TP도 pnl < -2%이면 손절 로직에 위임

### 수정 3: trigger_price 공식 수정
- **PROFIT_FLAT**: `current_price * 0.995` → `current_price * 1.002` (현재가 이상 보장)
- **LOSS_ZONE**: `avg_price * 0.99` → `avg_price * 1.005` (매입가 이상 보장)

### 수정 4: sell_targets 보호 시간 추가
- **executor.py `execute()`**: SIGNAL_EXIT는 매수 후 2시간 내 스킵

### 수정 5: 쿨다운 실패 시 보수적 처리
- `except: pass` → 실패 시 기본 4시간 쿨다운 적용, 매수 SKIP

### 수정 6: fallback 초단기 손절 기준 통일
- **horizon_manager.py**: fallback `stop_loss_pct` -0.7 → -1.5 (config와 일치)
