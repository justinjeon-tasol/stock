## 근본 원인 (3가지 복합)

### 1. EXIT_PLAN trigger_price가 매입가 미만에 설정됨
- `PROFIT_FLAT`: `s1 = max(avg_price * 1.005, current_price * 0.995)` → current_price * 0.995는 현재가보다 낮음
- `LOSS_ZONE`: `s1 = max(current_price * 1.02, avg_price * 0.99)` → avg_price * 0.99 = 매입가 -1%에서 발동 가능
- 결과: 손실 상태에서 "익절" 실행이라는 모순

### 2. sell_targets 경로에 보호 시간 없음
- WeightAdjuster가 SIGNAL_EXIT로 매도 지시하면 보유 1분이어도 즉시 실행
- `_check_stop_take`에는 2시간 보호가 있지만 sell_targets에는 없었음

### 3. 쿨다운 체크 실패 시 `pass`로 무시
- Supabase 조회 실패 시 쿨다운 4시간이 무효화 → 손절 직후 재매수 허용

## 수정 7: Plan-Your-Trade — exit_plan 매수 시 1회 확정 (커밋 ???)

### 구조적 문제
기존: 매 30분 사이클마다 `build_exit_plan()` 재실행 → forecast 변화에 따라 조건이 악화
- PROFIT_UP → PROFIT_FLAT → LOSS_ZONE으로 trend가 강등되면서 trigger_price 하향
- 수정 2~3에서 가드를 추가했지만 근본적으로 exit_plan이 흔들리는 구조

### 변경 내용
**원칙: 매수 시 TP/SL 확정, 이후 손절가는 상향만 허용**

#### A. 매수 시 exit_plan 즉시 생성 (`executor.py`)
- BUY 체결 직후 horizon_config 기반으로 고정 TP/SL 계산
- exit_plan DB 저장 (1회)
- 예: 단기 매수 @ 100,000 → TP1: +3% (103,000), TP2: +7% (107,000), SL: -3% (97,000)

#### B. orchestrator Step 2-c 덮어쓰기 차단 (`orchestrator.py`)
- 기존 exit_plan이 있으면 `build_exit_plan` 스킵
- exit_plan이 없는 포지션(레거시)만 생성

#### C. 트레일링 스탑 상향만 허용 (`executor.py`)

- 고점 갱신 → 손절가 상향 (기존 peak_price 로직 활용)
- 국면 급변(대폭락장) 시에만 긴급 오버라이드 허용

## 수정 8: 장기 포트폴리오 분리 관리

### 문제
기존 시스템은 모든 포지션을 단기 매매 파이프라인에서 관리.
장기 보유 종목(수개월~수년)이 30분 사이클 청산 로직에 걸려 조기 매도될 위험.

### 변경 내용

#### A. DB `portfolio_type` 컬럼 추가 (`database/db.py`)
- `positions` 테이블에 `portfolio_type` 컬럼 (기본값 `"short"`)
- `save_position()`에 `portfolio_type` 파라미터 추가
- `get_open_positions(portfolio_type=)`로 필터링 지원 (None이면 전체)
- 컬럼 미존재 시 전체 조회 폴백

#### B. 단기 파이프라인에서 장기 종목 제외 (`agents/position_manager.py`)
- `review_positions()`에서 `get_open_positions(portfolio_type="short")`만 조회
- 장기 종목은 30분 사이클 청산 로직에서 완전 분리

#### C. 장기 전용 체크 루프 (`main.py --mode long-check`)
- 1일 1회 실행용 (장 마감 후 권장)
- 장기 종목별 현재가 조회, 손익 계산
- exit_plan 없으면 자동 생성 (TP: +20%/+40%/+60%, SL: horizon_config 기반)
- 트레일링 스탑: 고점 갱신 시 SL 상향만 허용
- 손절/익절 도달 시 로그 경고 (수동 매도 검토)

#### D. 장기 종목 수동 등록 (`main.py --mode long-register`)
- 대화형 입력 (종목코드, 종목명, 수량, 매입단가)
- `portfolio_type="long"` 포지션 저장 + exit_plan 자동 생성

### 수정 파일
| 파일 | 변경 |
|------|------|
| `main.py` | `long-check`, `long-register` 모드 추가 |
| `database/db.py` | `portfolio_type` 파라미터, `get_open_positions` 필터링 |
| `agents/position_manager.py` | `review_positions`에서 `portfolio_type="short"` 필터 |
